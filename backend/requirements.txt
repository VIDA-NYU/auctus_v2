# Pinned to the transformers 4.x line so huggingface-hub stays <1.0, which keeps
# it compatible with AutoDDG's fastembed dependency (fastembed 0.5.1 requires
# huggingface-hub<1.0). The latest transformers 5.x requires huggingface-hub>=1.5
# and would conflict. Embedding model (all-MiniLM-L6-v2) behaves identically.

# On macOS-x86_64 (Intel) PyTorch caps at torch 2.2.2 (built for NumPy 1.x), and the
# pinned transformers/sentence-transformers above are from the NumPy-1.x era. There an
# unpinned NumPy 2.x crashes torch at import. Other platforms (Linux / Apple Silicon /
# Windows) get a NumPy-2-compatible torch, so they stay unconstrained.
numpy<2; sys_platform == "darwin" and platform_machine == "x86_64"
